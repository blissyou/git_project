1. Git 과 Github 의 역할에 대해 설명하시오.
>> Git - 로컬 저장소와 원격 저장소를 연결 및 컨트롤 할 수 있는 툴 
>> Github - 원격 저장소 역할

2. 터미널에서 현재 디렉토리를 변경할때 사용하는 명령어를 적으시오.
>> cd

3. 다음 사진은 Desktop 디렉토리에 git_test 디렉토리를 만든 후 이동한 상황이다.
이 폴더를 로컬 저장소로 사용하고 싶다면 어떤 명령어를 사용해야 하는지 적으시오
>> git init

4. 다음 사진은 로컬 저장소를 원격 저장소와 연결하는 명령어이다.(빈칸에 들어갈 단어를 적으시오)
>> remote add

5. 다음 사진은 파일 두 개를 만들고 어떤 명령어를 사용 후, git status를 사용하였을때의 화면이다.
어떤명령어를 입력했는지 완전하게 적으시오.
>> git add Quiz.txt

6. 다음 사진은 파일을 수정한 후 push 하였는데 에러가 뜬 상황이다.(에러 이유를 서술하시오)
>> github에 수정이 발생해서 로컬레포지토리 업데이트를 못함. git pull 로 해결

7. git branch의 장점을 설명하시오
>> 각 사용자 별로 나누어서 작업을 할 수 있고 동시에 공유할 수 있음. 개발 할때 처음부터 main에 올렸다
오류가 발생했을시 돌이킬수 없게 될 수도 있기때문에 이를 방지하기 위해 사용

8. 다음 사진은 현재 저장소에 있는 브랜치의 목록이다
현재 위치한 브랜치와 다른 브랜치로 이동하는 명령어를 적으시오
>> git checkout ONEZERO

9. 현재 브런치가 ONEZERO일때, main 브랜치에 변경사항을 적용시키는 명령어들을 순서대로 작성하시오.
(ONEZERO의 변경사항들은 push까지 완료한 상태임)
>> git checkout main
>> git merge ONEZERO
>> git push
